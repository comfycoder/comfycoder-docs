Docker Cheatsheet

Variables Shown In This Document

For the following Docker command examples, the variables show are defined as follows:

image_name = Name you wish to assign to the image (e.g., “myapp”)

image_tag = An image tag used to version a container. (e.g. “v1”, “12345”, or “1.2.3”) The goal is to produce a specific fixed tag to pin a container or generate an incremental tag to coerce an upgrade to a newer version of a container.

image_id = A randomly generated id value assigned to an image. (e.g., a3be3b32798e)

registry_name = (optional for local Docker containers) The fill name of a Docker or Private container registry. Note that DockerHub is the default container registry used by Docker on your local machine.

container_name = This is a combination of either of the following:

  • image_name:image_tag

(when running locally using docker)

  • registry_name/image_name:image_tag

(when referring to an image in a remote container registry)

container_id = A randomly generated id value assigned to a container.

port_int = The port number specified inside the docker file. (e.g., 80)

port_ext = The port address to externally expose an application when running a container inside docker. (e.g., 8081)

volume_name = The name of a volume (think directory folder) running inside of docker.

Docker Commands

List running containers

docker ps

List all containers (includes containers that are not running)

docker ps -a

Stop a container

docker stop container_name

docker stop container_id

Stop all containers

docker stop $(docker ps -aq)

Remove a container

docker rm -f container_name

docker rm -f container_id

Remove all containers

docker rm $(docker ps -aq)

List all images

docker images

docker image ls

Remove an image

docker rmi image_id

docker image rm image_name

Remove all images

docker rmi $(docker images -q) -f

Build a docker image

docker build -t container_name .

View docker container logs

docker logs container_name

docker logs container_id

Pull an image from a registry

docker pull registry_name/image_name

Run a container locally

docker run -d -p port_ext:port_int image_name

Run a container remotely

docker run -d -p port_ext:port_int –name container_name image_name

Run container using Git bash for local code in volume

docker run -d -p 8081:80 -v /$(pwd):/app registry_name/image_name

Run container using Windows command line for local code in volume

docker run -d -p 8081:80 -v $(pwd):/app registry_name/image_name

Docker Run Options

docker run

Additional command line arguments for Docker run command:

-d (run detached)

-it (interactive terminal

-p (publish a port)

-v (mount a volume)

–name (name a container)

–link (communicate between containers)

–rm (auto-delete when container stops)

-e NAME=value (environment varIables)

docker volume ls
